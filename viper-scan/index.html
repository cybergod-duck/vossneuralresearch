How They Do It — The Attack Chain

STAGE 2 — hCaptcha Cryptocurrency Proof-of-Work

hCaptcha doesn't just verify you're human — it runs a Proof-of-Work (PoW) computation that uses your CPU to solve cryptographic hash puzzles. The script contains references to blockchain terminology: hashNonce, prevHash, proofOfWork. Your machine is literally mining while you think about what song to generate. The computation runs inside a Web Worker so it doesn't trigger the browser's "page is unresponsive" warning — it burns CPU silently in the background.

STAGE 3 — Microsoft Clarity Session Replay → GPU Death Loop

This is the one that warps your windows. Microsoft Clarity records every mouse movement, scroll, click, and DOM mutation for "analytics." It does this by hooking MutationObserver to watch every element in the page, then serializing the entire DOM state on every change. On a complex page like Suno (with waveform visualizers, animation loops, and real-time playback controls), this creates a feedback loop:

1. Suno's UI animates a waveform → DOM mutation
2. Clarity observes the mutation → serializes DOM
3. Serialization changes DOM state → new mutation
4. GOTO 2 → infinite loop
5. Each loop forces a GPU compositor repaint
6. GPU saturates → window geometry corrupts → "weird shapes"

The GPU compositor is responsible for rendering ALL windows on the desktop, not just Chrome. When one Chrome renderer process saturates the GPU with repaints, it starves every other window of compositor time — causing desktop-wide geometry corruption. That's why your entire screen warps, not just the browser tab.

STAGE 4 — Incognito Bypass via SharedWorker

Even if a user opens Suno in Incognito mode, the tracker stack can persist identification through SharedWorker and BroadcastChannel APIs. These APIs allow communication between tabs — including between a normal tab and an incognito tab in certain browser configurations. Combined with browser fingerprinting (canvas hash, WebGL renderer string, audio context fingerprint, installed fonts), the tracker stack can identify the same user across sessions without cookies. Incognito provides zero meaningful protection.

STAGE 5 — The Long-Running Tab Problem (9.6 GB Profile Bloat)

Users who keep a Suno tab open for hours (common during long generation sessions) accumulate massive amounts of tracker state in the browser profile. Clarity alone can write hundreds of megabytes of session replay data to IndexedDB and Local Storage. Combined with hCaptcha's PoW cache, Braze's push notification tokens, and Criteo's retargeting pixel data, a single Chrome profile can bloat from ~200 MB to 9.6 GB over weeks of use. This data persists even after clearing browsing history because it lives in SQLite databases and LevelDB files in the profile directory — not in the browser's "clear data" scope.

STAGE 6 — HP AI for Print: The Amplifier

HP silently installs this extension via Windows Group Policy during printer driver installation. It has "Read and change all your data on all websites" permission plus webRequest / webRequestBlocking — meaning it intercepts every HTTP request on every page. Its 2.9 MB background.js contains 45 DoubleClick references and connections to Criteo, Tapad, Maze, and Suno's own tracking subdomain. This extension amplifies the tracker stack by injecting additional tracking into pages that wouldn't otherwise have it. Even pages with zero trackers get a tracking payload from this extension. It re-installs itself via registry policy even after manual removal — which is why we had to nuke the registry entry.
